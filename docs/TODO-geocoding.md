# GPS逆ジオコーディング改善

## 現状の問題

CLGeocoder でレート制限が発生し、同じフォルダの連続写真でも地名が取れたり取れなかったりする。

## 根本原因 (Gemini調査結果)

1. **ja/enの2回呼び出し = 2リクエスト扱い** (サーバー側でカウント)
2. **毎回プロセス起動 → CLGeocoderの内部キャッシュが効かない** (同一座標でも毎回サーバー問い合わせ)
3. **2秒間隔は攻めすぎ** (1分50回が閾値、ja/en2回で実質1秒1回)
4. **レート制限時にApple側が厳しくなる** (10秒バックオフでは不十分、60秒~5分必要)

## 改善方針

### 1. Swiftヘルパーを常駐プロセスに変更

毎回プロセス起動/終了ではなく、Rustから`spawn`してstdin/stdoutでやり取り。

メリット:
- CLGeocoderの内部キャッシュが効く (同一/近接座標はサーバー問い合わせなし)
- プロセス起動オーバーヘッドなし
- Swift側でリクエスト待ち行列を精密制御可能

### 2. ロケール取得を1回に削減

ja_JP で1回だけ取得。CLPlacemarkは多くの情報を持つので、英語地名はRust側でマッピング (都道府県名の英訳テーブル)。
→ APIリクエスト数を1/2に削減。

### 3. 同一座標のスキップ

Rust側で直前の座標との距離を計算。5メートル以内なら前回の結果を再利用。
連続写真(同じ場所で撮影)で30%以上のリクエスト削減。

### 4. レート制限時のバックオフ強化

- 空結果 → 最低60秒待機
- 連続失敗 → 5分待機
- CLError.network を検出してレート制限と通信エラーを区別

### 5. リクエスト間隔

1回のリクエスト後、最低2秒待機 (1分30回以下に抑える)。
